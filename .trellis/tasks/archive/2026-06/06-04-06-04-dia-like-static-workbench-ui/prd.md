# Dia-like static workbench UI slice

## Goal

Move the static validator workbench closer to the Dia-like layout using existing
`ProjectValidationIndex` truth only.

## Requirements

- Keep `ProjectValidationIndex` and `ValidationReport` as the only
  PASS/WARN/ERROR truth sources.
- Present a denser Dia-like workbench:
  - left side: device/group index with filter and document/profile status;
  - left lower area: validation summary and issue cards;
  - right side: selected component detail.
- Validated detail panels expose two explicit views:
  - `报告` for deterministic pin/model/compliance/evidence/summary sections;
  - `原理图连接` for netlist-derived schematic topology and scope boundaries.
- Component group rows may navigate to an existing validated refdes detail when
  one exists in that group. Manual/no-profile groups remain coverage rows and
  must not become electrical verdicts.
- The output remains a single static HTML file with no server dependency and no
  API key.

## Acceptance Criteria

- [x] Multi-component batch UI includes the left combined device/validation
      panel and right-side `报告` / `原理图连接` tabs.
- [x] Project workbench generated by `design-validator-ui` keeps existing
      validation counts unchanged.
- [x] Group rows with a validated refdes can select the matching detail panel;
      unmatched groups stay manual/coverage-only.
- [x] Evidence tokens, trust labels, document source tokens, and scope boundary
      wording remain visible in HTML.
- [x] Focused UI renderer/CLI tests pass.
- [x] Browser QA confirms the generated static workbench loads locally and tab
      switching/row selection works.
- [x] `uv run pytest -q`, `uv run ruff check .`, and `git diff --check` pass
      before completion.

## Notes

This is a display slice. It does not change validators, datasheet profiles,
document matching, candidate matching, or AI/Copilot behavior.
