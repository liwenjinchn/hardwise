# V1.1 Document Index Candidate Builder

## Summary

V1 proved the project-level loop: real Allegro/PST import, 132 component groups, local public document matching, and one MPQ8626 power-family validation path. V1.1 should make the next scaling step narrower and repeatable: generate a reviewable `document-index` draft from grouped coverage data so new public projects can quickly move from `no_result` to human-reviewed datasheet/document coverage.

The feature should help the user build a local/public datasheet index. It must not validate more devices, auto-download PDFs, or extract structured profiles yet.

## Goal

Given a `design-validator-ui` JSON sidecar with `component_groups`, produce a candidate document index draft for groups that are likely to need datasheets. The output should be easy for a human to review, edit, and then feed back into `design-validator-ui --document-index`.

## Product Shape

- Main entry remains `hardwise` CLI.
- Add a narrow command such as `build-document-index-candidates <validation-index.json>`.
- Input is an existing project index JSON generated by `design-validator-ui --index-json`.
- Output is CSV or TSV with one row per candidate group.
- Rows include enough audit context to review quickly:
  - identity / normalized_identity;
  - identity_kind;
  - suggested_family;
  - manufacturer if available;
  - refdes_count and refdes_sample;
  - current document_status and profile_status;
  - suggested search query;
  - blank `Title`, `URL`, `Path`, and `Notes` columns for human fill-in.

## Candidate Rules

- Include active/IC-like groups first: `ic`, `diode`, `transistor`, `inductor`, `ferrite`, and other non-passive families with usable identity.
- Exclude obvious passives by default: resistor/capacitor passive-value groups should not become datasheet search rows unless an explicit flag is later added.
- Exclude connector/test-point/mechanical groups by default.
- Exclude groups already `document_status=matched` unless an explicit future flag asks for all rows.
- Do not fabricate MPNs from refdes, footprint, or placeholder library names.
- Preserve `no_result`, `ambiguous`, and `manual_needed` so the draft explains why a row needs review.

## Output Contract

Default CSV columns:

```text
MPN,Manufacturer,Title,URL,Path,Description,IdentityKind,Family,RefdesCount,RefdesSample,DocumentStatus,ProfileStatus,SearchQuery,Notes
```

`MPN` should be filled only when the identity looks like a real or part-like device identity. For passive values and placeholders, leave it blank or exclude the row by default.

`SearchQuery` should be deterministic, for example:

```text
<identity> datasheet
<identity> <manufacturer> datasheet
```

No network calls in this slice.

## CLI Behavior

- Default output path can be derived from input stem, e.g. `reports/<project>-document-candidates.csv`, with `--output` override.
- CLI summary should print input rows/groups, candidates written, skipped passive/mechanical counts, and matched-doc groups skipped.
- Malformed JSON, missing `component_groups`, or unreadable input are real errors and should exit non-zero.

## Tests

- Unit test candidate filtering from a small synthetic project index payload.
- CLI test using an existing fixture/generated JSON sample or a temp JSON file:
  - writes CSV;
  - includes IC-like no-result rows;
  - excludes resistor/capacitor passive values by default;
  - excludes already matched MPQ8626 rows by default;
  - output has stable columns.
- Regression smoke can use the real-project JSON manually, but tests must not depend on the user's local Allegro folder path.

## Non-Goals

- No online search.
- No automatic datasheet download.
- No PDF parsing.
- No datasheet fact extraction.
- No new datasheet profile.
- No second family validator.
- No PCB/layout/PLM/supplier/lifecycle features.

## Done When

- A project index JSON with 132 groups can produce a concise candidate CSV for the next document-index review pass.
- The output can be edited into the existing `document-index` format and used with `design-validator-ui --document-index`.
- `uv run pytest -q` and `uv run ruff check .` pass.
