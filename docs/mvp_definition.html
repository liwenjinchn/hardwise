Hardwise MVP Definition

Hardwise 的 MVP 是一个可信的 pre-Layout 原理图评审工作台。它不证明 LLM 能独立判断完整硬件设计；它证明一个更窄、更有用的流程：公开 EDA 文件、BOM 身份和公开 datasheet/profile 证据进入评审工作台，每个对象可追溯，每个不确定项留给 reviewer。

Review NodeLayout handoff 前的 schematic review。
Trust Boundaryrefdes 来自 EDA registry；finding 必须带 source token。
AI RoleCopilot 解释工具证据，不替代 deterministic verdict。

User Problem

目标用户是准备 schematic 进入 Layout handoff 的硬件工程师。评审前他们需要快速知道：哪些器件必须看、哪些只是 manual gap、每条结论由哪一页 datasheet、哪条 netlist fact、哪条 rule 或哪行 BOM 支撑，以及 AI 有没有提到真实存在的板上对象。

Core Flow

  1. 导入公开 schematic project，或 schematic netlist + BOM。
  2. 建立 refdes、pin、net、BOM identity 的可信 EDA registry。
  3. 匹配本地公开 datasheet index 和 reviewed structured profiles。
  4. 运行 deterministic checklist rules 和 component-family validators。
  5. 把输出分成 Must reviewManual gapPassed。
  6. Copilot 只解释 tool-backed facts：component context、validation result、topology、datasheet evidence、unknown-refdes miss。
  7. 导出 review workbench 和反馈列表，供 schematic review meeting 逐项关闭。

UI Shape

  • Project summary: input files, component count, BOM match, validated count, PASS/WARN/ERROR/manual counts。
  • Review queue: issue-first 的 Must Review / Manual Gap / Passed rows。
  • Component detail: identity、pins、nets、validation result、profile/document state、evidence tokens。
  • Copilot panel: 展示 tool trace 和 trust tier，解释结果但不覆盖 L1 verdict。
  • Feedback export: refdes、描述、证据、reviewer 接受/拒绝、close status。

Scope

In scope: public KiCad project、public Allegro netlist/PST + schematic BOM fixture、registry-verified component/pin/net/BOM identity、public datasheet/profile evidence、deterministic rules、reviewed component validators、L1/L2/L3 trust tiers、static HTML workbench 和本地 Copilot panel。

Non-Goals

不做 PCB layout、.brd、boardview、placement/routing、SI/PI、EMC、thermal、simulation、PLM、supplier risk、price/inventory、production BOM governance、公司内部硬件数据、整板自动正确性判断、profile draft 自动转 ready。

Acceptance

  • 公开 demo input 能打开含 project summary、review queue、component detail、evidence 和 manual gap 的工作台。
  • 未知位号如 U999 不能被展示成有效板上对象。
  • 每个 hard finding 都有 source token；没有证据就降级。
  • 至少一个 L1 deterministic issue 能从 component 追到 net/profile evidence。
  • 至少一个 L2 datasheet answer 展示 datasheet:<file>#pN token。
  • Copilot answers 暴露 tool trace，不能覆盖 L1 validator result。
