Hardwise 设计验证器 Demo

把项目里的可验证问题先摊开

输入公开 Allegro schematic netlist 和 schematic BOM，Hardwise 自动匹配本地 structured profile，生成项目级器件索引、验证摘要和器件级报告。未匹配 profile 的器件会显式留在 manual/no-profile 列表里，不伪装成已经验证。

它解决什么

  • 评审前先知道：哪些器件已验证，哪些器件还缺 profile。
  • 把 DCDC、gate driver、MCU debug/startup 这类 schematic-side 问题变成确定性检查。
  • 让工程师先看问题器件，而不是在整张 BOM 里翻找。

输出长什么样

  • 一个本地静态 HTML 工作台：左侧器件索引，中间摘要，右侧详情。
  • 一个项目级 markdown / JSON index，记录 validated、manual、no-profile。
  • 每个已验证器件都有 pin summary、外围/拓扑检查和可下载报告。

为什么可信

  • 结论来自 netlist + BOM identity + structured profile，不让模型自由判硬件。
  • 位号仍经过 registry guard，证据和 scope boundary 会随报告输出。
  • 未知二极管、电压、拓扑不会被硬判为 PASS；会落到 WARN 或 manual。

截图级核心工作流

这版没有做上传账号和次数体系，先把本地项目打开后的核心体验收紧：器件列表、文档/profile 状态、验证摘要、问题详情、报告下载和范围声明。

U12 / XL1509

OUTPUT 接到 D5 和 L1，但 D5 是 1N4007W 普通整流二极管，L1 是 6.8uH，低于 profile 推荐范围。

结论

U12 ERROR：续流二极管和电感选择都会进入外围/拓扑检查，而不是塞进某个 pin 的自由文本里。

U3 / EG2132

VB/VS bootstrap 路径存在，但 D1=MBRA210LT3G 属于低耐压线索，在 24V bus fixture 中判为风险。

结论

U3 ERROR：半桥 gate driver 的自举二极管、电容、HO/LO gate load 都按 deterministic topology rule 检查。

U8 / STM32G030

PA13/SWDIO 被接到 SWCLK，PA14/SWCLK 被接到 SWDIO。

结论

U8 ERROR：本轮只做 MCU basic 的 power、reset、BOOT0、SWD 检查，不做固件、clock tree 或完整 AF matrix。

工作台里的问题卡片示例

硬件工程师看到的是验证语言：哪个器件、哪个外围或脚位、为什么失败、下一步该替换或确认什么。

ERROR U12 / XL1509-12E1 D5 + L1

D5=1N4007W 不符合 XL1509 buck 续流二极管的 Schottky-style 要求；L1=6.8uH 低于 profile 推荐最小电感量。

建议替换为符合电压/电流裕量的肖特基二极管，并按 24V 输入、12V 输出重新选择电感范围。

证据位置：datasheet:xl1509.pdf#p9 · bom:mixed_controller_power_stage#D5/L1
1. 接入 EDA 数据

解析公开 Allegro netlist 和 schematic BOM，建立项目器件索引。

2. 匹配本地 profile

按 BOM identity 匹配 L78、XL1509、EG2132、STM32G030。

3. 运行确定性检查

把 DCDC、gate driver、MCU SWD/BOOT/RESET 检查输出成结构化结果。

4. 留下可追溯结果

输出静态 HTML、markdown index、JSON sidecar 和 scope boundary。

一句话亮点

这个 demo 展示的不是“让 AI 自由评审硬件”，而是把 schematic netlist、BOM identity、structured datasheet profile 接成可审计的 deterministic validation workbench；LLM 只在 registry/evidence guard 的边界内解释。

uv run hardwise design-validator-ui tests/fixtures/allegro/mixed_controller_power_stage.net tests/fixtures/allegro/mixed_controller_power_stage_bom.csv --output reports/controller-design-validator.html --index-output reports/controller-design-validator-index.md --index-json reports/controller-design-validator-index.json
