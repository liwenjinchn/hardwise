前端界面变迁记录

a69c1c4 单器件 静态 HTML

阶段 1：简陋但已经有证据意识

最早的 UI 是一个单器件 validator。它能展示 components、nets、selected、status，也有 pin validation 和 datasheet token，但还需要手动指定 refdes 与 profile，信息架构更像工程调试报告。

  • 入口是 `report-validator-ui`，一次只围绕一个器件展开。
  • 左侧是 component index，右侧是 U1 的 pin-level 验证表。
  • 已经出现防编造项目的核心材料：状态、net、summary、evidence token。
uv run hardwise report-validator-ui ... U1 data/datasheet_profiles/l78.json
c53d0d0 批量验证 左表 + 顶部指标 + 右详情

阶段 2：开始像工作台，支持多个问题排队

批量 UI 把多个被验证器件放到同一屏，顶部出现 validated/errors 指标，右侧详情可以在 U1、U12 之间切换。这个阶段已经接近你说的“左上/左下/右边”的中间形态。

  • 从单点检查变成多个目标的 review queue。
  • 错误器件可以和通过器件并列，评审优先级开始浮现。
  • 视觉上仍是静态报告风格，交互与产品导航还比较弱。
uv run hardwise report-validator-ui-batch ... U1=... U12=...
87045cc tabs/detail ERROR 队列

阶段 3：detail 区被产品化，评审动作更明确

这个节点变成深色“原理图检验工具”：左侧是器件队列，右侧是选中器件详情，顶部 metrics 更密集，详情页出现报告、原理图连接等 tab。它第一次有了明显的工作台气质。

  • 选择器件、查看报告、跳转连接这些动作形成固定结构。
  • PASS/WARN/ERROR 与 L1 确定性标签在视觉上更突出。
  • 仍然是单页静态 UI，但已经从“表格报告”转向“审查界面”。
uv run hardwise report-validator-ui-batch ... --targets-manifest ...
87045cc 项目级 自动匹配

阶段 4：从器件验证页升级为项目工作台

`design-validator-ui` 自动匹配 profile，生成项目级 index，并可嵌入 AI snapshot。左侧不再只是器件表，而是按 BOM 身份、类别、资料状态组织的评审对象，右下角出现 AI 入口。

  • 指标从 selected/errors 扩展到 BOM matched、validated、manual 等项目级状态。
  • 资料缺口和人工确认成为界面一等公民。
  • 这是静态离线快照和 live workbench 之间的桥。
uv run hardwise design-validator-ui ... --ai-snapshot
current React SPA 方向 SaaS 产品化

阶段 5：多标签 SaaS 化评审台

当前界面已经形成完整产品骨架：顶部导航包含导入、解析、审查、AI 助手、问题清单、导出；左侧是 review queue，中间是器件详情和 task list，右侧是 evidence trace。它更像一个可反复使用的 SaaS 工具，而不是一次性 HTML 报告。

  • 导航、状态芯片、队列筛选、详情卡、证据栏和 Copilot 入口形成稳定信息架构。
  • 错误、证据、人工判断、外部提示被拆成不同工作区，评审流程更可操作。
  • 视觉从早期的 report printout 转为密集但清晰的 operational SaaS。
uv run hardwise design-validator-ui ... --ai-snapshot --output /tmp/hardwise-ui-evolution/html/05-current-saas-workbench.html

这条演进线说明了什么

界面不是单纯变漂亮，而是逐步把审查工作的对象、状态和证据关系产品化。

从 refdes 到队列早期只能解释一个器件，后期能把 25 个器件按风险和状态组织成 review queue。
从表格到任务PASS/WARN/ERROR 不再只是统计，变成“必须修、看证据、人工判断”的工作项。
从证据 token 到证据栏最早的 datasheet token 最终发展成右侧 evidence trace，让每条结论可追溯。
从静态报告到产品顶部多标签和 Copilot 入口把导入、解析、审查、问答、导出串成完整闭环。
