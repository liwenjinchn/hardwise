60-90 second demo script

Hardwise Demo 录屏脚本

主线只讲 trust architecture：模型不能自由评审硬件，必须被 EDA registry、evidence token、deterministic validator 和 Refdes Guard 约束住。coverage 数字只当支撑材料。

录屏前准备

uv sync
uv run hardwise review data/projects/pic_programmer --rules R001,R002,R003,DS001 --report-style component --output /tmp/hardwise-review.md
uv run hardwise design-validator-ui \
  tests/fixtures/allegro/mixed_controller_power_stage.net \
  tests/fixtures/allegro/mixed_controller_power_stage_bom.csv \
  --ai-snapshot \
  --output /tmp/hardwise-copilot-workbench.html

分镜

01
README 开场，10 秒

Hardwise 不是让 LLM 自由判断硬件，而是一个 pre-Layout 原理图评审工作台。模型只能通过结构化工具拿到 EDA registry、netlist 和 evidence token，输出再经过 Refdes Guard。

02
工作台摘要，15 秒

指顶部指标：25 components、17 L1 rows、PASS/WARN/ERROR=5/9/3。强调 17 个 L1 rows 包括 4 个 profile-backed targets 和 13 个 generic passive checks，剩余 8 个保持 manual/no-local-profile。

03
确定性问题，20 秒

点 U12，展示 buck 错误。这个 ERROR 不是模型意见，而是 deterministic validator 从网表和 XL1509 profile 得到的结论。

04
Copilot trace，20 秒

打开右下角 AI 按钮，点已烘焙问题。展示 tool trace：run_component_validation 是 L1 deterministic，证据 token 必须可见。

05
Refdes Guard，15 秒

点“板上有没有 U999?”。不存在的位号不会被模型编造成器件，最终显示会把它包成 ⟨?U999⟩。

06
收束，10 秒

只做 schematic-side review，只用公开输入，不做 PCB layout、boardview、PLM、供应商、价格库存，也不碰公司内部硬件数据。

不要这么说

  • 不说“模型独立验证整块板”。
  • 不说 generic passive checks 是深度 datasheet review。
  • 不说所有 profile token 都经过 live vector retrieval。
  • 不暗示覆盖 PCB geometry、layout、routing、supplier status、lifecycle 或 PLM。
