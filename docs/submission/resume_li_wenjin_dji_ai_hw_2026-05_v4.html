Hardwise — 硬件原理图评审 AI Agent
2026.05 · github.com/liwenjinchn/hardwise
独立设计 · 主导落地 · Python 3.11 · Anthropic SDK · Pydantic · KiCad · Chroma · SQLAlchemy 2.0 (SQLite default, PostgreSQL via HARDWISE_DB_URL)
面向硬件原理图评审节点的 AI Agent:读取公开 KiCad 工程与器件资料,自动生成 evidence-token 可追溯的 HTML 评审报告,减少人工查资料、对位号、整理结论的重复工作。
  • 流程拆解 · 端到端跑通:在公开 KiCad 工程 pic_programmer 上跑通解析 → 检索 → 规则检查 → 报告生成,解析 121 器件、77 NC 引脚,生成 84 条带 evidence-token 评审发现(人工 1–2 天 → Agent 分钟级)。
  • EDA 二次开发:独立设计 KiCad S-expression 解析适配层(不依赖 kiutils),把原理图结构化为可被 Agent 查询的 EDA registry;抽象 EDA adapter 边界,为后续 Cadence/Allegro 接入预留扩展点。
  • 双存储 · 向量 + 结构化:Chroma 存 datasheet 切页向量,SQLAlchemy 2.0 存器件/网络/规则(SQLite default + PostgreSQL override),以 refdes 位号作为跨存储 join key。
  • 反幻觉两层守卫:工具层对未知 refdes 返回 {found: false, closest_matches};出站文本经正则扫描 + EDA registry 二次校验,未命中 token 自动包裹为 ⟨?Xnnn⟩,评审意见必须携带可校验 evidence token;命中 datasheet 时追加 datasheet:<pdf>#p<N> 来源。
  • Agent 工程化:Anthropic SDK tool-use 循环 + Pydantic 校验工具 I/O;三档模型路由(fast / normal / deep)从 env 装配;Sleep Consolidator 把会话沉淀为人工 gate 的候选规则。
项目用 AI 编程工具辅助实现;架构决策、反幻觉守卫与验证策略由本人独立设计、审查与对答。所有 demo 输入均为公开 KiCad 工程与公开 datasheet,不涉及任何雇主内部数据。
