01

先建项目索引

从 schematic netlist 和 BOM 里拿到真实位号、器件值、MPN 和连接关系，未匹配 profile 的器件不会被隐藏。

02

跑确定性验证

当前覆盖 L78、XL1509、EG2132、STM32G030 basic；问题来自 profile + topology，不是模型自由发挥。

03

保留边界和证据

报告保留 datasheet/profile token 和 scope boundary，明确不做 PCB、PLM、供应链或固件判断。

面试时可以这样讲

直接展示输入、输出和边界：哪些判断已经 deterministic，哪些器件仍然是 manual/no-profile。

面试官它怎么发现 U12 的问题？
Hardwise先匹配 XL1509 profile，再从 netlist 看到 OUTPUT 接 D5 和 L1，确定 D5=1N4007W 不是 Schottky-style，L1=6.8uH 低于推荐范围。
面试官它是不是自动评审全 BOM？
Hardwise不是。当前只验证有本地 profile 的 4 个器件；其余 21 个在 project index 中显式显示为 no_profile/manual。
