One-sentence pitch

Hardwise does not ask a model to freely review hardware; it turns schematic netlist, BOM identity, and structured datasheet profiles into an auditable deterministic validation workbench, while the LLM stays behind registry/evidence guards.

#001需确认
器件：U12profile：XL1509状态：ERROR

OUTPUT topology contains D5=1N4007W and L1=6.8uH. The XL1509 profile requires a Schottky-style freewheel diode and a larger inductor range for the 24V-to-12V fixture.

建议替换续流二极管并按 profile 重新选择电感；同一工作台还会显示 U3 bootstrap diode 和 U8 SWDIO/SWCLK 互换错误。

01Parse design

从公开 netlist + BOM 构建 component、net、BOM registry。

02Match profiles

本地 catalog 匹配 L78、XL1509、EG2132、STM32G030。

03Run checks

电源、DCDC、gate driver、MCU basic 规则输出结构化 validation。

04Guard output

未命中 registry 的位号会被包成 `⟨?U999⟩`，结论带 evidence/scope。

05Write workbench

生成单文件 HTML、markdown index 和 JSON sidecar。

uv sync
uv run hardwise design-validator-ui \
  tests/fixtures/allegro/mixed_controller_power_stage.net \
  tests/fixtures/allegro/mixed_controller_power_stage_bom.csv \
  --output reports/controller-design-validator.html \
  --index-output reports/controller-design-validator-index.md \
  --index-json reports/controller-design-validator-index.json
