Hardwise / Phase 4 + C5 closeout

Five mechanisms, three trust tiers

Hardwise 的头条不是“让模型更会说”，而是把硬件 Agent 输出压进五个 trust 机制和 L1/L2/L3 分层。KiCad 与 Allegro 两条公开输入轨负责展示同一条主干的不同侧面。

Five Mechanisms

01Refdes Guard

用户可见位号必须来自 KiCad registry 或 Allegro netlist/BOM identity。

02Evidence Ledger

finding 没有 source token 就不能进入最终报告。

03Sleep Consolidator

重复问题先进入人工审核候选规则，不自动污染规则库。

04Tiered Routing

模型按 fast / normal / deep env slot 选择，代码不写死 vendor model。

05Prompt Caching

静态 agent prompt 可缓存，并有实测 cache-read 命中。

Boundary: C3/C4 coverage ranking is supporting material, not the headline. The headline is the trust architecture: five mechanisms, L1/L2/L3 evidence tiers, registry-verified objects, deterministic validators, and structured tools. Phase 4 does not add PCB/boardview, placement/routing, simulation, PLM, supplier/lifecycle/pricing, hosted app state, or company-internal hardware data.
