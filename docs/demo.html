进 Layout 前，先把风险排成队

Hardwise 的头条不是“让模型更会说”，而是把公开原理图或 schematic netlist+BOM 变成评审者能直接使用的队列：必须评审、人工补证、已通过和证据。五个 trust 机制与 L1/L2/L3 分层是这条工作流的安全边界。

L1 / L2 / L3 证据分层

层级含义当前可见证据
L1 确定性Python rule / validator 决定 PASS/WARN/ERROR；模型只解释结构化结果。run_component_validation 和工作台 validated rows。
L2 有页码证据本轮 datasheet search 返回带 source_pdf + page 的检索证据，供 reviewer 核验；不是逐句语义证明。L78 快照 trace 显示 datasheet:l78.pdf#p4。
L3 人工确认没有 ready profile 或没有 retrieval evidence，系统保持人工确认。no-profile/manual rows；无 vector hit 的 datasheet 问答。
主证据轨KiCad不需要 API key
