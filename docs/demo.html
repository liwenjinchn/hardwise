One-sentence pitch

Hardwise does not ask a model to “look smart”; it constrains the model with KiCad-derived tools, then publishes every claim with a source token so hardware reviewers can audit the output quickly.

C2 value field '220uF' does not declare rated voltage. The 80% derating rule cannot be evaluated without it.

建议补充耐压后缀，例如 `100uF/25V`，再由评审人核对实际工作电压。

Evidence token

sch:pic_programmer.kicad_sch#C2
01Parse KiCad

从公开工程构建 component / pin / NC registry。

02Run checks

R001/R002/R003 输出结构化 Finding，而不是自由文本。

03Guard refdes

未命中 registry 的位号会被包成 `⟨?U999⟩`。

04Gate evidence

没有 `sch:` / `datasheet:` / `rule:` token 的结论不进报告。

05Write report

生成 markdown / HTML 报告和 SQLite 结构化库。

uv sync
uv run hardwise review data/projects/pic_programmer \
  --rules R001,R002,R003 \
  --format html

# output
report: reports/pic_programmer-YYYYMMDD.html
store:  reports/pic_programmer.db

Demo 输入只使用公开 KiCad 样例和公开 datasheet。项目边界锁定在 pre-Layout schematic review；PCB review、PLM、FMEA 和公司内部硬件数据都不进入这个 MVP。
