固定录屏顺序

Step屏幕动作讲清楚的点
项目摘要顶部 metrics 和 grouped review queue。输入已经变成 review workbench，不是聊天记录。
Must Review切到 Must Review 区域。ERROR/WARN 在 Layout handoff 前保持显眼。
U12 deterministic ERROR打开 XL1509 detail。ERROR 来自 netlist + reviewed profile 的 Python validator。
Copilot trace展开 baked Copilot 的 Evidence / Tool trace。Copilot 解释 structured tool result，不决定 PASS/WARN/ERROR。
U999 wrapped点“板上有没有 U999?”。未知位号返回 structured miss，并显示成 ⟨?U999⟩。
L78 evidence token点 L78 evidence 问题或 U1/L7805 trace。datasheet:l78.pdf#p4 是可见、可核对的 L2 evidence token。

可信边界

01位号护栏

用户可见位号必须来自 parsed EDA registry。

02证据账本

finding 没有 source token 就不能进入最终报告。

03规则候选池

重复问题先进入人工审核候选规则，不自动污染规则库。

04模型分层路由

模型按 fast / normal / deep env slot 选择，代码不写死 vendor model。

05Prompt 缓存

静态 agent prompt 可缓存，并有实测 cache-read 命中。

L1 / L2 / L3 证据分层

层级含义可见证据
L1 确定性Python rule / validator 决定 PASS/WARN/ERROR；模型只解释结构化结果。run_component_validation 和 workbench validated rows。
L2 有页码证据本轮 datasheet search 返回带 source_pdf + page 的检索证据，供 reviewer 核验。L78 trace 显示 datasheet:l78.pdf#p4。
L3 人工确认没有 ready profile 或没有 retrieval evidence，系统保持人工确认。no-profile/manual rows；无 vector hit 的 datasheet 问答。
主舞台exported netlist/PST+BOM离线 Copilot

pic_programmer：registry / L78 / ask 复现

  "absolute maximum input voltage" \

边界：coverage ranking 是支撑材料，不是主角。主角是 review workflow：registry-verified 对象、确定性 validator、evidence token、可见 manual gap，以及结构化 Copilot 工具。MVP 不包含实时 EDA 插件、PCB/boardview、placement/routing、仿真、PLM、supplier/lifecycle/pricing、hosted app state 或公司内部硬件数据。
